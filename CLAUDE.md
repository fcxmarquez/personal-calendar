## Agentic Workflow

Your development process will be as follows:

1. Develop phase. This is your standard model development process
2. Review phase. You will invoke the review agent to review the code and provide feedback. After receiving the review agent's feedback, use the skill check-review and address the addressed issues.
3. Test phase (optional). If the code involve frontend or product development. You will invoke the test agent to test the code and provide feedback. You will receive the test agent's feedback and then address any addressable issue.
4. Commit phase. You will invoke the commit skill to commit the changes to the remote repository.

This phase workflow will be followed ONLY and EXCLUSIVELY once you determine to start the development process of a new feature or a bug fix. If you are not sure about the feature or bug fix, you should not start the development process.
